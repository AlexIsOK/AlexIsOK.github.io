How to setup a Minecraft server for free on dedicated hardware.

This is assuming you already have a Linux server up and running.

You will need:

  • At least 4GB of memory dedicated to the server, or 10GB for large servers.
  • At least 3GB of storage dedicated to the server, or 50GB for large servers.
  • A network connection with a ping of less than 100ms to 1.1.1.1 and fast-ish speeds.
  • Access to the server via SSH and FTP (or any protocol that transfers files).
  • Any Linux distro that you can easily install Java 11 on.

Setting up

Make sure you have OpenJDK 11 installed, you can use anything that's Java 8+, but OpenJDK seems to be faster.
For this tutorial, I recommend using Paper's server software rather than Mojang's default one, as it is much faster and can handle plugins.

You can download Paper here, or Mojang's server here.
You can create a virtual environment to run the server in for better security if you're installing plugins, but you don't need it otherwise.



Running the software

After the software is downloaded into it's own directory (e.g. /home/minecraft/server/), go into the directory and type java -jar server.jar (or whatever the name is), when it closes, "read the server software eula in eula.txt" and agree to it, then run it again. Besides port forwarding port 25565 (tcp and udp), the server is all setup!! you can download plugins for servers from Spigot MC and Bukkit, both will work for paper, just make sure that the Minecraft version listed for the server matches the version you're running, otherwise it could cause some compatibility issues and long stack traces in the terminal ;-;



Installing plugins

Want plugins? It's super simple. All you need to do is send the jar to the path of the minecraft server in the plugins directory, something like /home/minecraft/server/plugins/, then just restart the server!



Optimizations

There is a page on optimizing Minecraft server stuff here that fits most users needs.



WARNINGS!!!

Do NOT name the startup script for the server start.sh, it will 100% cause issues. Name it server.sh or something instead.
